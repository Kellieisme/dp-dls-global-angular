# Sidesheet Component

### API references
The `ba-sidesheet` component is a custom Atmosphere component designed to provide a fly-out side sheet panel from the left or right side of the viewport. It is built using Angular and Angular Material primitives such as `mat-sidenav`, `mat-toolbar`, and `mat-button`. The sidesheet can be used to present additional options, context, or actions without navigating away from the main content.

This component supports two variants: `standard` (side mode) and `modal` (over mode), controlling how the sidesheet interacts with the page content. It supports configurable width, position (`start` or `end`), and an optional footer with customizable buttons.

## Directives

### SideSheet Component
Selector: `<ba-sidesheet>`

#### Usage

```typescript
import { Component } from '@angular/core';
import { SideSheetComponent, SidesheetVariant, SidesheetPosition } from './sidesheet.component';

@Component({
  selector: 'app-example',
  standalone: true,
  imports: [SideSheetComponent],
  templateUrl: './example.component.html',
  styleUrls: ['./example.component.scss'],
})
export class ExampleComponent {
  sidesheetOpened = false;

  buttons = [
    { text: 'Cancel', buttonType: 'text', onClick: () => this.sidesheetOpened = false },
    { text: 'Save', buttonType: 'flat', onClick: () => alert('Saved!') },
  ];

  variant = SidesheetVariant.STANDARD;
  position = SidesheetPosition.START;
}
```

```html
<ba-sidesheet
  [opened]="sidesheetOpened"
  [buttons]="buttons"
  [variant]="variant"
  [sidesheetPosition]="position"
  [showFooter]="true"
  title="Example SideSheet"
>
  <!-- Custom content can be projected here if supported -->
</ba-sidesheet>

<button mat-button (click)="sidesheetOpened = true">Open SideSheet</button>
```

## Properties

<Markdown>
{`
| Name | Description |
|------------------------------|-----------------------------------------------|
| @Input() buttons: AtmosphereSidesheetButtonsArray | Array of buttons to render in the footer. Each button supports 'text', 'onClick' callback, and optional 'buttonType' ('text', 'stroked', 'flat'). |
| @Input() opened: boolean | Controls whether the sidesheet is open ('true') or closed ('false'). Defaults to 'false'. |
| @Input() showFooter: boolean | Whether to show the footer section with buttons. Defaults to 'true'. |
| @Input() sidesheetWidth: string | CSS width of the sidesheet panel. Defaults to '320px'. |
| @Input() sidesheetPosition: 'start', 'end' | Position of the sidesheet on the viewport. Defaults to 'start'. |
| @Input() title: string | Title text displayed in the sidesheet toolbar. Defaults to empty string. |
| @Input() variant: AtmosphereSidesheetVariant | Variant of the sidesheet: 'standard' (side mode) or 'modal' (over mode). Defaults to 'standard'. |
| readonly sidesheetMode: AtmosphereSidesheetContentMode | Internal getter that returns 'side' for standard variant and 'over' for modal variant. |
`}
</Markdown>

## Troubleshooting

### Sidesheet does not open or close:
Ensure the opened input is bound to a boolean property and updated properly. Use Angular event bindings or methods to toggle this property.

### Buttons in footer do not respond:
Verify that each button in the buttons array has a valid onClick function defined.

### Sidesheet width not applying:
Confirm that sidesheetWidth is a valid CSS width string (e.g., '320px', '25rem').

### Positioning issues:
Use only 'start' or 'end' for sidesheetPosition. Other values are not supported.

### Footer not showing:
Check that showFooter is set to true (default). If set to false, footer and buttons will be hidden.
