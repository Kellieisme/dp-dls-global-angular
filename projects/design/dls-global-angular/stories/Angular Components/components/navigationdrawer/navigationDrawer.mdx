# Navigation Drawer

The `NavigationDrawerComponent` is a versatile sidebar navigation component built using Angular Material. It provides a collapsible and expandable navigation drawer that can be presented in different styles, including default, standalone, and modal. The component supports dynamic menu items, allowing for a structured navigation experience within applications.

### Selectors

- `ba-navigation-drawer`: The main selector for the navigation drawer component.

### Usage

In your typescript file.

The selector, templateUrl, styleUrls, and class name (NavigationDrawerPageComponent) can be different based on your project structure and naming conventions.

```typescript
import { Component } from '@angular/core';
import { NavigationDrawerComponent } from '@design/dls-global-angular/navigation-drawer';
import { UserProfileComponent } from '@design/dls-global-angular/user-profile';
import { MatMenuModule } from '@angular/material/menu';
import { MatDividerModule } from '@angular/material/divider';
import { IconRegistryModule } from '@design/dls-global-angular/icon-registry';
import { MatIconModule } from '@angular/material/icon';
import { MatButtonModule } from '@angular/material/button';

@Component({
  selector: 'navigation-drawer-page',
  standalone: true,
  imports: [NavigationDrawerComponent, UserProfileComponent, MatMenuModule, MatButtonModule, MatDividerModule, IconRegistryModule, MatIconModule],
  templateUrl: './navigation-drawer-page.component.html',
  styleUrl: './navigation-drawer-page.component.scss'
})
export class NavigationDrawerPageComponent {

}
```

## API reference

## Features
- Customizable branding (app name & logo)
- Supports different presentation styles: `Default`, `Standalone`, and `Modal`
- Configurable expansion and collapse behavior
- Slot for user profile and notifications button

Important Note on the opened Property
The opened property of the NavigationDrawerComponent is used to control the visibility of the navigation drawer. To effectively manage this property, you should implement two-way data binding. This allows the parent component to update the opened state based on user interactions, including clicks on the backdrop.

Two-Way Binding Example
To enable two-way binding for the opened property, you can use the following syntax in your template:

```html
<ba-navigation-drawer
  [appLogo]="'my-logo'"
  [appName]="'My Application'"
  [homeRoute]="'/'"
  [navDrawerMenuItems]="myMenuItems"
  [navDrawerPresentation]="'standalone'"
  [(opened)]="opened"> <!-- Two-way binding for opened state -->
</ba-navigation-drawer>
```
